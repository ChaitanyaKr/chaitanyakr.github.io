Code Search with Named Entities

Team size: 2
Advisor: Dr. Srikantha Bedathur, Dr. Venkatesh Vinayakrao
Course: Information Retrieval
Concepts/Technologies: Named Entities, Entity Linking, Entity Profiling


Source code contains syntactic redundancies which are usually discussed using natural language terms. For example, the syntax “(int) a” is referred to as type casting in natural language. Now, if a person searches for type-cast in this code, he/she will not find this statement because the terms are different.
This project uses the questions and answers in StackOverflow to capture such relations between syntax and the terms developers use to refer to them.

Co-Drive collaborative storage

Team size: 4
Advisor: Dr. Rahul Purandare
Course: Practice of Programming
Concepts/Technologies: Android, Parse, SonarQube, JavaScript


Often offline storage runs out, and the user then needs to store data either on cloud, or on external media. We propose a solution, wherein the user needn’t be concerned about the security of their data in the cloud, by simply storing it onto a dedicated space on one of her trusted peers, within the network.
Another use case could be sharing of media with another peer, via that shared space.
We have implemented an application that allows multiple users to share some space on each of their drives (offline storage). This is achieved by initially setting up a fixed amount of space on each user’s local drive, required for the purpose of the application. Once a user issues a request to his list of “friends”, it can be accepted for storing the user’s data.

Commit sizes

Team size: 1
Advisor: Dr. Rahul Purandare
Concepts/Technologies: Static Analysis, Software Engineering, Version Control


Collaborative software development allows developers to work in parallel, making them (as a team) more efficient, but it introduces conflicts in the process. This work attempts to alleviate the problem of conflicts in such collaborative environments by a dependency analysis approach, applied to source code.
Secondly, it attempts to see if large enough commits exist such that they could be broken down into smaller units, thereby reducing chances of merge conflicts.

CryptAnalyzer

Team size: 1
Advisor: Dr. Somitra Sanadhya
Concepts/Technologies: Static Analysis, Android, Cryptography, Soot, Flowdroid


Developers use cryptographic APIs in Android with the intent of securing data such as passwords and personal information on mobile devices. However, often they lack the necessary background to assist them choose the right parameters for such APIs. Consequently, method calls are often set to their default values. However, these may not always be the most secure configuations.
CryptAnalyzer is a static analysis tool that checks Android APKs for six cryptographic properties borrowed from the paper below. These cover block cipher modes, initialization vectors and seeds to random generators.

This work was based on the paper Egele, M., Brumley, D., Fratantonio, Y., & Kruegel, C. (2013, November). An empirical study of cryptographic misuse in android applications. In Proceedings of the 2013 ACM SIGSAC conference on Computer & communications security (pp. 73-84). ACM.
